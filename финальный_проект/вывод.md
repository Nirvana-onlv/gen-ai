# Финальный проект

# Отчёт: Автоматический анализ жалоб покупателей Wildberries

---

## 1. Задача и данные

**Задача.** Построить мультиагентный пайплайн, который на входе принимает негативный отзыв покупателя с Wildberries, а на выходе даёт: структурированный разбор проблемы по аспектам, предполагаемы ответ продавца и автоматическую оценку его качества. Этот агент - попытка решить проблему продавцов WB с ответом на многочисленные и зачастую однотипные отзывы.

**Данные.** Датасет [`nyuuzyou/wb-feedbacks`](https://huggingface.co/datasets/nyuuzyou/wb-feedbacks) на HuggingFace — отзывы с Wildberries на русском языке. Поля: текст отзыва (`text`), оценка покупателя (`productValuation`, 1–5), ответ продавца (`answer`).

**Подготовка.** Стриминг через `datasets` (HF), фильтры:
- оценка 1–3 (только негативные/смешанные)
- текст ≥ 20 символов
- ответ продавца ≥ 20 символов (пустые записи исключены — они бесполезны для RAG)

Итого: 970 записей для основного корпуса, 30 отложено для валидации.

---

## 2. Система

**4 техники курса:**

| Техника | Реализация |
|---|---|
| Information Extraction + аспектный анализ + Map-Reduce | Agent Analyst извлекает аспекты (quality, delivery, packaging, compliance, price, service) с цитатами |
| RAG | Chroma + эмбеддинг-модель `MiniLM-L12-v2`; индекс из реальных ответов продавцов на отзывы покупателей; топ-3 похожих примера передаются Writer-у как контекст для дальнейшего ответа |
| Мультиагент | Три агента последовательно: Analyst → Writer → Judge |
| LLM-as-judge | Agent Judge оценивает сгенерированный ответ по трём осям (вежливость, конкретность, решение проблемы) и сравнивает с оригиналом |

**Схема пути данных:**
```
wb-feedbacks (HF streaming)
    → prepare_input.py: фильтр (rating≤3, answer непустой) → input/reviews_eval.jsonl
    → rag.py: build_index() → Chroma (эмбеддинги ответов продавцов)
    → [Agent Analyst]: IE + аспектный анализ → AspectAnalysis
    → [Agent Writer]: аспекты + RAG топ-3 → SellerReply
    → hallucination.py: сверка ответа с отзывом → HallucinationReport
    → [Agent Judge]: LLM-as-judge → JudgeVerdict
    → output/eval_results.jsonl + output/eval_table.csv
```

## 4. Где пайплайн ошибался

### Пример 1 — выдуманные сроки (отзыв №11)

**Вход:** «Где деньги за возврат?»

**Ответе Writer-а:** «Обычно деньги поступают в течение 5–10 рабочих дней после принятия товара на складе.»

**Проблема:** покупатель не уточрнил большую часть информации касаемо возврата. Модель не может знать за какое время осуществляется возврат и какое конкретно решение на текущем этапе взаимодействия с покупателем принял продавец по возврату денег. Следовательно, ответ является шаблонным и выдуманым со стороны агента. 

### Пример 2 — ложные обещания (отзыв №14)

**Вход:** «Сама лампа огонь, но второй день подряд привозят на пункт лампу б/у. Это ужас. Одну и туже будто возят...»

**Галлюцинации в ответе Writer-а:**
- «мы вернём вам деньги в полном объёме»
- «готовы выслать вам новую лампу после проверки её состояния»

**Проблема:** покупатель не просил ни возврата денег, ни замены — он описал ситуацию. Агент взял на себя обязательства по возврату денег, хотя не наделен такими полномочиями и не может обещать такого решения. 

### Пример 3 — слабый аспектный анализ коротких отзывов (отзывы №1, №22, №30)

**Вход №1:** «Бальзам не понравился. Простой вазелин с легким запахом мяты.»

**Проблема:** Агент не смог ответить на этот отзыв.

## 5. Выводы

Пайплайн работает — 93% сгенерированных ответов оценены Judge как лучше оригинальных ответов продавцов из датасета. Средний score 4.20/5.

5 галлюцинаций в 4 из 30 отзывов. Все четыре случая имеют схожий характер Writer придумывает конкретные обязательства (сроки, обещания возврата/замены) когда отзыв касается сервисных вопросов, а не качества товара. На отзывах про качество и упаковку галлюцинаций не было.

**Что оставить в прод:**
- Структуру мультиагента (Analyst → Writer → Judge) — она работает и легко расширяется
- Проверку галлюцинаций как обязательный шаг перед отправкой ответа покупателю
- RAG на ответах продавцов — retrieval стабильно находит 3 релевантных примера

**Что переделать:**
- Добавить фильтр в Writer: агент не может обещать тех вещей, о которых он не имеет данных, либо не наделен полномочиями. Необходимо четче определить его круг обязанностей.
- Улучшить обработку коротких отзывов, поскольку сейчас агент часто теряется на них и не может выдать валидный ответ.
